NAME
        ps - report a snapshot of the current processes.
        对当前的进程输出一份快照

SYNOPSIS
        ps [OPTIONS]

DESCRIPTION
        ps displays information about a selection of the active processes. If you want a repetitive update of the selection and the displayed information, use top(1) instead.

        ps 的功能，是选定一些活动进程，然后将这些进程的信息打印出来。 
        如果想要看的是持续更新的进程信息，使用 top 命令。

        This version of ps accepts several kinds of options:

            1   UNIX options, which may be grouped and must be preceded by a dash.
            2   BSD options, which may be grouped and must not be used with a dash.
            3   GNU long options, which are preceded by two dashes.

        Options of different types may be freely mixed, but conflicts can appear. There are some synonymous options, which are functionally identical, due to the many standards and ps implementations that this ps is compatible with.

        Note that "ps -aux" is distinct from "ps aux". The POSIX and UNIX standards require that "ps -aux" print all processes owned by a user named "x", as well as printing all processes that would be selected by the -a option. If the user named "x" does not exist, this ps may interpret the command as "ps aux" instead and print a warning. This behavior is intended to aid in transitioning old scripts and habits. It is fragile, subject to change, and thus should not be relied upon.

        ps 接受以下三种风格的选项：
            1. UNIX 方式，亦即跟在 '-' 之后，可以组合在一起。（这是 ps 认为的标准方式）
            2. BSD 方式，亦即单独的选项，可以组合在一起
            3. GNU 方式，亦即跟在 '--' 之后

        三种风格的选项有可能可以混用，也有可能存在冲突。
        三种风格中的某些选项，可能具有完全相同的功能，这是因为 ps 具有众多的标准，ps 为了保存对不同的标准的实现，也是为了保护兼容性，

        注意，'ps -aux' 是与 'ps aux' 不同的。
        'ps -aux' 是 UNIX 风格，功能是打印出所有 x 用户的进程的信息，同时打印出所有被 -a 选项选定的信息。
        但是，如果当 x 用户不存在，则 ps 会将命令按照 'ps aux' 进行识别并执行，并打印出处一个告警。
        这种行为是为了对旧有的脚本和习惯的兼容，这种情况非常不牢靠，且有可能改变，应当尽量避免。

        By default, ps selects all processes with the same effective user ID (euid=EUID) as the current user and associated with the same terminal as the invoker. It displays the process ID (pid=PID), the terminal associated with the process (tname=TTY), the cumulated CPU time in [DD-]hh:mm:ss format (time=TIME), and the executable name (ucmd=CMD). Output is unsorted by default.

        The use of BSD-style options will add process state (stat=STAT) to the default display and show the command args (args=COMMAND) instead of the executable name. You can override this with the PS_FORMAT environment variable. The use of BSD-style options will also change the process selection to include processes on other terminals (TTYs) that are owned by you; alternately, this may be described as setting the selection to be the set of all processes filtered to exclude processes owned by other users or not on a terminal. These effects are not considered when options are described as being "identical" below, so -M will be considered identical to Z and so on.

        Except as described below, process selection options are additive. The default selection is discarded, and then the selected processes are added to the set of processes to be displayed. A process will thus be shown if it meets any of the given selection criteria.

        默认情况下，ps 选取的进程是所有属于当前用户的并且属于当前终端的进程。
        默认打印的信息包括：PID、TTY、累积 CPU 时间、程序名。
        默认不排序。

        如果使用的是 BSD 风格的选项，则会额外添加进程状态 STAT、执行程序的参数等信息。
        这个特征可以通过 PS_FORMAT 环境变量进行覆盖。
        并且，使用 BSD 风格的选项，默认选取的进程为所有属于当前用户的进程，不管是否属于当前终端。（也就是说，从全部进程中剔除属于其他用户的进程和不属于任何终端的进程。）
        BSD 风格选项的这种特性，即使与其他风格的选项的功能被描述为'完全相同'时，也存在。
        例如，虽然认为 '-M' 和 'Z' 两个选项功能相同，但实际上 'Z' 选项会额外输出 BSD 风格选项的那些额外信息。

        除非额外说明，否则进程选取的选项都是加法性的。（可以一次性指定多个，选取的进程为所有结果的并集）
        当使用了任意进程选取选项时，则不再按照默认方式选取进程，而是会按照选项的要求选取进程。

EXAMPLES

        To see every process on the system using standard syntax:
        使用标准方式，查看系统上的所有进程
            ps -e
            ps -ef
            ps -eF
            ps -ely

        To see every process on the system using BSD syntax:
        使用 BSD 风格，查看系统上的所有进程
            ps ax
            ps axu

        To print a process tree:
        查看进程树
            ps -ejH
            ps axjf

        To get info about threads:
        查看线程信息
            ps -eLf
            ps axms

        To get security info:
        查看安全信息
            ps -eo euser,ruser,suser,fuser,f,comm,label
            ps axZ
            ps -eM

        To see every process running as root (real & effective ID) in user format:
        查看所有以 root 身份运行的进程，无论是 RUID 还是 EUID，并且输出信息遵循 user-format 格式
            ps -U root -u root u

        To see every process with a user-defined format:
        查看所有进程，但输出信息按照用户自定义的格式
            ps -eo pid,tid,class,rtprio,ni,pri,psr,pcpu,stat,wchan:14,comm
            ps axo stat,euid,ruid,tty,tpgid,sess,pgrp,ppid,pid,pcpu,comm
            ps -Ao pid,tt,user,fname,tmout,f,wchan

        Print only the process IDs of syslogd:
        只打印所有 syslogd 进程的 pid 信息
            ps -C syslogd -o pid=

        Print only the name of PID 42:
        只查看 pid 为 42 的进程的程序
            ps -q 42 -o comm=

OPTIONS
    SIMPLE PROCESS SELECTION 进程选择基础部分


        -A, -e
            Select all processes. 
            选择所有进程。

        -a
            Select all processes except both session leaders (see getsid(2)) and processes not associated with a terminal.
            选择所有属于当前终端的进程，但排除 session leaders 进程。

        -d
            Select all processes except session leaders.
            选择所有进程，但排除 session leaders 进程。


        -N, --deselect
            Select all processes except those that fulfill the specified conditions (negates the selection).
            反向选择进程。
        
        （以下为 BSD 风格）

        ax
            a: Lift the BSD-style "only yourself" restriction, which is imposed upon the set of all processes when some BSD-style (without "-") options are used or when the ps personality setting is BSD-like. The set of processes selected in this manner is in addition to the set of processes selected by other means. An alternate description is that this option causes ps to list all processes with a terminal (tty), or to list all processes when used together with the x option.
            x: Lift the BSD-style "must have a tty" restriction, which is imposed upon the set of all processes when some BSD-style (without "-") options are used or when the ps personality setting is BSD-like. The set of processes selected in this manner is in addition to the set of processes selected by other means. An alternate description is that this option causes ps to list all processes owned by you (same EUID as ps), or to list all processes when used together with the a option.
            选择所有进程，BSD 风格。
            （a、x 虽然可以分开使用，但基本都是组合在一起使用）
            （单独使用 a 表示挂起 BSD 风格的 'only yourself' 限制，仅保留 'must have a tty' 限制，效果为选择所有属于某个终端的进程）
            （单独使用 a 表示挂起 BSD 风格的 'must have a tty' 限制，仅保留 'only yourself' 限制，效果为选择所有当前用户的进程）
            （单独使用时，均可以将选择的结果与其他选项选择的结果进行汇总）
            （没有必要单独使用，可以使用 UNIX 风格的相应的选项）

        g
            Really all, even session leaders. This flag is obsolete and may be discontinued in a future release. It is normally implied by the a flag, and is only useful when operating in the sunos4 personality.
            （不翻译，已被废弃，后续版本可能会删除，功能基本可以被 'a' 选项替代）

        T
            Select all processes associated with this terminal. Identical to the t option without any argument.
            选择所有属于当前终端的进程。
            功能与 't' 选项不加任何参数时相同。

        r
            Restrict the selection to only running processes.
            仅选择 running 状态的进程

    PROCESS SELECTION BY LIST 按照列表选择进程

        These options accept a single argument in the form of a blank-separated or comma-separated list. They can be used multiple times. For example: ps -p "1 2" -p 3,4
        下文中所说的列表 'list'，可以是空白间隔的列表，也可以是逗号间隔的列表，并且不同形式的列表也可以多次出现。

        -123, -p pidlist, --pid pidlist
            Select by PID. This selects the processes whose process ID numbers appear in pidlist. 
            按照 pid 选择进程
        
        -C cmdlist
            Select by command name. This selects the processes whose executable name is given in cmdlist.
            按照程序名选择进程。
        

        -g grplist
            Select by session OR by effective group name. Selection by session is specified by many standards, but selection by effective group is the logical behavior that several other operating systems use. This ps will select by session when the list is completely numeric (as sessions are). Group ID numbers will work only when some group names are also specified. See the -s and --group options.
            按照 session 或 EGID 来选择进程。
            （逻辑上讲，应该是只按照有效组来选择进程，但有些标准指定了按照会话来选择进程，因此才让此选项同时具有两个功能）
            如果 'grplist' 中全是数字时，则按照 session 选择进程。
            如果 'grplist' 中至少存在一个组名时，才会按照 EGID 来选择进程。

        --group grplist
            Select by effective group ID (EGID) or name. This selects the processes whose effective group name or ID is in grplist. The effective group ID describes the group whose file access permissions are used by the process (see getegid(2)). The -g option is often an alternative to --group.
            按照 EGID 选择进程。
            'grplist' 中可以是 ID，也可以是 name。
            （所谓有效组，就是指进程使用的此组的文件访问权限。）
            通常 -g 选项是 --group 选项的替代方案。
        
        -G grplist, --Group grplist
            Select by real group ID (RGID) or name. This selects the processes whose real group name or ID is in the grplist list. The real group ID identifies the group of the user who created the process, see getgid(2).
            按照 RGID 选择进程。
            'grplist' 中可以是 ID，也可以是 name。
            （所谓 RGID，是用来创建进程的用户的分组。更多可以查看 getgid 的帮助）
        
        --ppid pidlist
            Select by parent process ID. This selects the processes with a parent process ID in pidlist. That is, it selects processes that are children of those listed in pidlist.
            按照父进程的 pid 来选择进程。

        -q pidlist, --quick-pid pidlist
            Select by PID (quick mode). This selects the processes whose process ID numbers appear in pidlist. With this option ps reads the necessary info only for the pids listed in the pidlist and doesn't apply additional filtering rules. The order of pids is unsorted and preserved. No additional selection options, sorting and forest type listings are allowed in this mode. Identical to q and --quick-pid.
            按照 pid 来选择进程，不过采用快速模式。
            所谓快速模式，是指 ps 只会读取进程的必要信息。
            不会进行额外的过滤规则。结果不排序，而是按照在 pidlist 中的顺序。
            基本上只能单独使用，不能与其他的进行选取选项、排序选项、森林选项一起使用。
            
        -s sesslist, --sid sesslist
            Select by session ID. This selects the processes with a session ID specified in sesslist.
            按照 session 选择进程。
            
        -t ttylist, --tty ttylist
            Select by tty. This selects the processes associated with the terminals given in ttylist. Terminals (ttys, or screens for text output) can be specified in several forms: /dev/ttyS1, ttyS1, S1. A plain "-" may be used to select processes not attached to any terminal.
            按照 tty 选择进程。
            'ttylist' 中的项的格式可以是：/dev/ttyN, ttyN, N。
            如果 '-' 作为 'ttylist' 中唯一的项，则表示选择不属于任何终端的进程

        -u userlist, --user userlist
            Select by effective user ID (EUID) or name. This selects the processes whose effective user name or ID is in userlist. The effective user ID describes the user whose file access permissions are used by the process (see geteuid(2)). Identical to U and --user.
            按照 EUID 选择进程。
            'userlist' 中可以是 ID，也可以是 name。
            （所谓进程的有效用户，是指进程使用的是此用户的文件访问权限）
            
        -U userlist, --User userlist
            Select by real user ID (RUID) or name. It selects the processes whose real user name or ID is in the userlist list. The real user ID identifies the user who created the process, see getuid(2).
            按照 RUID 选择进程。
            'userlist' 中可以是 ID，也可以是 name。
            （所谓进程的真实用户，是指创建进程的用户）
        
        （以下为 BSD 风格）
        
        123, p pidlist
            Select by process ID. Identical to -p and --pid.
            （不翻译，与 -p 功能相同）

        q pidlist
            Select by process ID (quick mode). Identical to -q and --quick-pid.
            （不翻译，与 -q 功能相同）

        t ttylist
            Select by tty. Nearly identical to -t and --tty, but can also be used with an empty ttylist to indicate the terminal associated with ps. Using the T option is considered cleaner than using t with an empty ttylist.
            （不翻译，与 -t 功能相同，或者说基本完全相同）

        U userlist
            Select by effective user ID (EUID) or name. This selects the processes whose effective user name or ID is in userlist. The effective user ID describes the user whose file access permissions are used by the process (see geteuid(2)). Identical to -u and --user.
            （不翻译，与 -u 功能相同）

    OUTPUT FORMAT CONTROL 控制输出信息的格式

        These options are used to choose the information displayed by ps. The output may differ by personality.
        以下选项用来指导 ps 输出的信息中包含哪些列。
        不同的风格的 ps 相互之间可能不同

        -c
            Show different scheduler information for the -l option.
            显示进程的调度策略 CLS。

        --context
            Display security context format (for SELinux).
            显示进程的安全文本信息 CONTEXT。

        -f
            Do full-format listing. This option can be combined with many other UNIX-style options to add additional columns. It also causes the command arguments to be printed. When used with -L, the NLWP (number of threads) and LWP (thread ID) columns will be added. See the c option, the format keyword args, and the format keyword comm.
            在输出信息时，输出更详细的信息。
            可能某一列的内容会比之前更详细，如在输出 command 时，内容中会添加参数。
            可能会额外输出更多的列，如 '-L' 选项与 '-f' 一起使用时，会额外添加 PPID、NLWP 等列。

        -F
            Extra full format. See the -f option, which -F implies.
            与 '-f' 功能类似，但比 '-f' 更详细。
            
        -j
            Jobs format.
            按照作业的格式进行输出。
            会增加 PGID、SID 两列。
        
        -l, -y
            -l: Long format. The -y option is often useful with this.
            -y: Do not show flags; show rss in place of addr. This option can only be used with -l.
            -l 选项表示按照长格式进行输出。
            会额外添加 F、S、UID、PPID、C、PRI、NI、ADDR、SZ、WCHAN 这几列
            -y 选项只能配合 -l 选项一起使用，表示在 -l 输出的列中，去掉 'F' 这一列
            
        -M
            Add a column of security data. Identical to Z (for SELinux).
            额外添加 LABEL 列，表示与 selinux 安全有关的信息，通常被用作 selinux 的安全性文本。

        -o format, --format format
            User-defined format. format is a single argument in the form of a blank-separated or comma-separated list, which offers a way to specify individual output columns. The recognized keywords are described in the STANDARD FORMAT SPECIFIERS section below. Headers may be renamed (ps -o pid,ruser=RealUser -o comm=Command) as desired. If all column headers are empty (ps -o pid= -o comm=) then the header line will not be output. Column width will increase as needed for wide headers; this may be used to widen up columns such as WCHAN (ps -o pid,wchan=WIDE-WCHAN-COLUMN -o comm). Explicit width control (ps opid, wchan:42,cmd) is offered too. The behavior of ps -o pid=X,comm=Y varies with personality; output may be one column named "X,comm=Y" or two columns named "X" and "Y". Use multiple -o options when in doubt. Use the PS_FORMAT environment variable to specify a default as desired; DefSysV and DefBSD are macros that may be used to choose the default UNIX or BSD columns.
            在输出信息时，按照用户指定的格式输出。
            'format'，是一个 blank 或 comma 分隔的列表，让用户可以指定输出哪些列。
            'format' 中支持的关键字，在 'STANDARD FORMAT SPECIFIERS' 一节中有详细的说明。
            可以通过 'ps -o Keyword=Value' 的方式，对列标题重命名。
            如果某一个 Keyword 为空，如 'ps -o Keyword01= -o Keyword02=Value02'，则此列的列表标题会被省略。
            如果所有的 Keyword 都为空，则列标题行会被省略。
            修改列标题的一个好处，是列的宽度会随着列标题的宽度增长而变长，因而可以通过修改列标题来对列进行增宽。
            ps 也提供了显式控制列标题的语法，格式为 'ps -o Keyword:Width'
            使用 'Keyword=Value' 的格式时需要注意，对于诸如 'ps -o Keyword01=X,Keyword02=Y' 时的表现，不同的 ps 风格会有不同的输出。
            有的会输出一列，列标题为 'X，Keyword02=Y'，有的会输出两列，第一列的列标题为 'X'，第二列的列标题为 'Y'。
            因此，如果存在疑问，则可以多次使用 '-o' 选项，每一个 '-o' 选项只接一个 Keyword。
            PS_FORMAT 环境变量，可以用来设置 ps 命令的默认输出格式。
            DefSysV 和 DefBSD 是两个宏，可以用来选择默认的 UNIX 或 BSD 列。
            
        -O format
            Like -o, but preloaded with some default columns. Identical to -o pid,format,state,tname,time,command or -o pid,format,tname,time,cmd, see -o above.
            与 '-o' 类似，但比 '-o' 的优点在于默认已经加载了一些比较基础的列，不需要在命令行上再指定这些基础信息列了。
            预加载的列包括：pid、state、tname、time、command，或者 pid、tname、time、cmd。
            'format' 部分会被放在 'pid' 之后。

        （以下为 BSD 风格）
        
        j
            BSD job control format.
            按照 BSD 作业控制的格式进行输出。
            输出信息时，会增加 PPID、PGID、SID、TPGID、STAT、UID 这几列，同时 COMMAND 列会连同命令的参数一起输出

        l
            Display BSD long format.
            按照 BSD 长格式进行输出。
            会增加 F、UID、PPID、PRI、NI、VSZ、RSS、WCHAN、STAT 这几列，同时 COMMAND 列会连同命令的参数一起输出

        o format
            Specify user-defined format.  Identical to -o and --format.
            （不翻译，与 -o 功能相同）

        O format
            is preloaded o (overloaded). The BSD O option can act like -O (user-defined output format with some common fields predefined) or can be used to specify sort order. Heuristics are used to determine the behavior of this option. To ensure that the desired behavior is obtained (sorting or formatting), specify the option in some other way (e.g. with -O or --sort). When used as a formatting option, it is identical to -O, with the BSD personality.
            （不翻译，此选项太过冗杂，又是能指定 format，又是能指定排序）
            （题外话，此选项可以说明为什么我不喜欢 BSD 格式的选项）
            （Heuristics，启发法。他妈的 option 的作用应该是清晰并且被严格限定范围的，操他妈的启发法）

        s
            Display signal format.
            按照信号的格式进行输出。
            会额外增加与信号有关的列，包括 UID、PENDING、BLOCKED、IGNORED、CAUGHT、STAT 这几列。

        u
            Display user-oriented format.
            按照面向用户的格式进行输出。
            会额外增加这几列：USER、%CPU、%MEM、VSZ、RSS、STAT、START。

    NOTES
        1. This ps works by reading the virtual files in /proc. This ps does not need to be setuid kmem or have any privileges to run. Do not give this ps any special permissions.
        此版本的 ps，是通过阅读 /proc 中的文件，来进行工作的。
        对于 ps 程序，不需要进行 setuid，不需要添加任何额外的权限。

        2. This ps needs access to namelist data for proper WCHAN display. For kernels prior to 2.6, the System.map file must be installed.
        （不懂）

        3. CPU usage is currently expressed as the percentage of time spent running during the entire lifetime of a process. This is not ideal, and it does not conform to the standards that ps otherwise conforms to. CPU usage is unlikely to add up to exactly 100%.
        对于 cpu usage 的计算方法，是用进程 running 的时间除以进程的整个生命的时间。
        这种计算方法并不理想，也不符合 ps 应当遵循的表针，
        cpu usage 一般不太可能上涨到 100%。

        4. The SIZE and RSS fields don't count some parts of a process including the page tables, kernel stack, struct thread_info, and struct task_struct. This is usually at least 20 KiB of memory that is always resident. SIZE is the virtual size of the process (code+data+stack).
        SIZE 列和 RSS 列通常不统计进程的某些特定的部分，如 page tables、kernel stack、struct thread_info、struct task_struct。
        这些内容通常占据大概 20K 的内存空间并且长驻。
        SIZE 是进程的虚拟大小，包括 code+data+stack。

        5. Processes marked <defunct> are dead processes (so-called "zombies") that remain because their parent has not destroyed them properly. These processes will be destroyed by init(8) if the parent process exits.
        如果一个进程的 STAT 列显示为 '<defunct>'，则表示此进程为僵尸进程。
        所谓僵尸进程，是指已经 dead，但其父进程并没有适当地将其 destroy。
        僵尸进程会在父进程 exit 时，被 init 所 destroy。

        6. If the length of the username is greater than the length of the display column, the numeric user ID is displayed instead.
        对于用户 name，如果超过了列宽，则会改而显示 ID。

        7. Commands options such as ps -aux are not recommended as it is a confusion of two different standards. According to the POSIX and UNIX standards, the above command asks to display all processes with a TTY (generally the commands users are running) plus all processes owned by a user named "x". If that user doesn't exist, then ps will assume you really meant "ps aux".
        对于可能引起两种风格的选项的混淆的命令，如 'ps -aux'，应当尽力避免使用。

ENVIRONMENT VARIABLES
        The following environment variables could affect ps.
        
		In general, it is a bad idea to set these variables. The one exception is CMD_ENV or PS_PERSONALITY, which could be set to Linux for normal systems. Without that setting, ps follows the useless and bad parts of the Unix98 standard.
		
		以下为 ps 的一些环境变量
		
		大多数情况下，不要修改这些环境变量。
		CMD_ENV 和 PS_PERSONALITY 有可能需要改为适合 linux，否则 ps 就会遵循 Unix98 标准，会很垃圾。
		

        COLUMNS
            Override default display width.
			设置 ps 的屏幕宽度

        LINES
            Override default display height.
			设置 ps 的屏幕高度

        PS_PERSONALITY
            Set to one of posix, old, linux, bsd, sun, digital... (see section PERSONALITY below).
			设置 ps 的风格，如 linux、posix 等

        CMD_ENV
            Set to one of posix, old, linux, bsd, sun, digital... (see section PERSONALITY below).
			设置 ps 的环境变量风格，如 linux 等

        I_WANT_A_BROKEN_PS
            Force obsolete command line interpretation.
			要求 ps 识别已废弃的命令

        LC_TIME
            Date format.
			设置 ps 的时间格式

        PS_COLORS
            Not currently supported.
			（目前不支持）

        PS_FORMAT
            Default output format override. You may set this to a format string of the type used for the -o option. The DefSysV and DefBSD values are particularly useful.
			设置 ps 的默认输出格式。
			可以设定为一个 '-o' 选项的 format 列表。
			DefSysV 和 DefBSD 是两个非常有用的值。
			
        PS_SYSMAP
        PS_SYSTEM_MAP
            Default namelist (System.map) location.

        POSIXLY_CORRECT
            Don't find excuses to ignore bad "features".
			（不懂）

        POSIX2
            When set to "on", acts as POSIXLY_CORRECT.
			（不懂）

        UNIX95
            Don't find excuses to ignore bad "features".

        _XPG

			（不懂）
